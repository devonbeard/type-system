Supernormal Web Design

A manifesto for supernormal websites

“An object becomes Super Normal through use. As designers we can aim at achieving the Super Normal by being less concerned with visual aspects of an objects character, by attempting to anticipate the objects likely impact on the atmosphere and how it will be to live with.”

—Jasper Morrison

A website should:

  • Load fast
  • Be readable
  • Be universally accessible
  • Use semantic markup
  • Use clear and concise language
  • Have a white background
  • Use a near black text color
  • Use system fonts
  • Use 16px font size body copy
  • Have bold weight headings
  • Use a reasonable measure
  • Have flush-left, ragged right text
  • Have underlined links
  • Use visible, persistent global navigation
  • Use standard form controls
  • Show labels for form elements
  • Keep focus states intact
  • Use links for navigation
  • Use buttons for forms and actions
  • Use compressed images where appropriate

A website should not:

  • Use webfonts
  • Load unnecessary images
  • Inhibit the default interaction behavior
  • Have superfluous animations
  • Hijack scrolling
  • Load background videos
  • Use interstitial modals
  • Autoplay media
